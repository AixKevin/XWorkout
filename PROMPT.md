# XWorkout 项目默认提示词

## 用户原始需求

```
我想开发一个android应用，应该采用什么现代化的技术栈？
```

### 详细需求
1. 这是一个健身记录软件，运行在用户端（client-only），主要功能是创建健身计划、记录健身情况
2. 支持语言为简体中文（zh-cn）
3. 支持平台：Android，测试用机为一加11（骁龙8 gen 2）。后续有跨平台想法
4. 体积精简，占用资源少，界面美观，采取现代化开发技术栈与ui审美（偏向apple风格）
5. 基础功能包括：
   - a. 创建健身计划，可自定义按几天一循环，支持设定周期内某天为训练日/休息日，并指定任意训练条目
   - b. 自定义训练条目（比如卧推、引体向上，可自定义条目名称），支持设定强度（可设定项包括：重量，次数等）
   - c. 可以在每个训练日添加任意数量指定训练条目
   - d. 记录运动实际情况，从训练计划导入，可输入实际完成强度等
   - e. 支持偷懒（请假）功能，将当前训练日改为休息日，并推迟一天训练日，支持撤销
6. 主要解决痛点（参照开源项目wger）：
   - a. 训练项目可自定义，不局限于预设的项目
   - b. 无需营养、图记等过多功能，仅需要核心的记录健身功能，要能按日期回顾等
   - c. 界面ui更加现代化、优美
   - d. 界面简洁，底部导航栏只需要三个按钮：今日（包括今日训练计划一览、偷懒等功能），训练（包括训练计划自定义、训练项目自定义、开始记录等功能），更多（包括设置，预留新功能位置）

## 技术选型

| 技术 | 选择 | 原因 |
|------|------|------|
| 框架 | Flutter | 跨平台、性能好、生态丰富 |
| 状态管理 | Riverpod | 简洁、功能强、测试友好 |
| 数据库 | Drift | 类型安全、SQLite封装 |
| 路由 | go_router | 声明式路由、类型安全 |
| UI风格 | Cupertino | Apple风格、现代化 |

## 开发规范

### 目录结构
```
lib/

│   ├── database/           # Drift 数据库
```

## Git 提交规范

### 每次更改后必须执行 git 提交

完成任何工作后，必须：

1. **查看更改状态**
   ```bash
   git status
   ```

2. **查看具体更改**
   ```bash
   git diff
   ```

3. **添加更改**
   ```bash
   git add .
   ```

4. **提交更改**
   ```bash
   git commit -m "简要的提交信息"
   ```

### 提交信息格式

使用简洁的中文提交信息：
- 添加新功能：`feat: 添加xxx功能`
- 修复问题：`fix: 修复xxx问题`
- 优化代码：`refactor: 优化xxx代码`
- 更新文档：`docs: 更新xxx文档`
- 环境配置：`chore: 配置xxx环境`

### 示例
```bash
git add .
git commit -m "feat: 添加训练项目管理功能"
```
